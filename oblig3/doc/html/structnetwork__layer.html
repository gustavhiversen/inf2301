◆ corrupt_prob

double corrupt_prob

Specifies the probability that the network layer instance will corrupt a package during transmission.

A 32-bit floating point number between 0.0f and 1.0f, where 0.0f specifies no corruption and 1.0f specifies certain data corruption.

double loss_prob

A 32-bit floating point number between 0.0f and 1.0f, where 0.0f specifies no package loss and 1.0f specifies certain package loss.

◆ remote_stack

osi_stack_t* remote_stack

A reference to the OSI-stack that represents the remote endpoint this network layer is connected to.

The value of this field is either a reference to the OSI-stack on the other side of the communication channel, or NULL if this network layer instance is not connected to any remote endpoint.

◆ transmit_callback

void(* transmit_callback) (void *ctx[3])

A reference to the callback function to call whenever a transport package is to be transmitted over the communcation channel.

A reference to a function that has no return value and takes an array-reference with 3 elements, where each element is a generic reference.

This field has a value of NULL if the network layer is not connected or if the connection has been torn down.

The callback function will receive the following values in its ctx argument:

IndexDetails
ctx[0]
osi_stack_t*		An ost_stack_t reference representing the destination OSI-stack.
ctx[1]
tick_timer_t*		A tick_timer_t reference to the timer that expired for the callback to be triggered.
ctx[2]
transport_package_t*		A transport_package_t reference to the transport package that is transmitted.

The callback function is responsible for deallocating ctx[1] using the free function, ctx[2] using the transport_pkg_destroy function, as well as the actual ctx reference using the free function.
