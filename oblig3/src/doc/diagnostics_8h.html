Detailed Description

Contains the diagnostic API to provide diagnostic printing to the standard output stream.
Students are not required to understand the implementation details of these utilities.

The declarations within this file provide developers with a utility function DIAG_PRINTF that provides a pre-formatted printf call with optional severity masking.

§ DIAG_PRINTF()

int DIAG_PRINTF ( diag_trace_lvl_t  severity,
osi_stack_t *  osi_stack,
diag_trace_src_t  layer,
const char  fmt[],
  ... 
)

Submits a pre-formatted printf call to the tracing system. The TRACE_LEVEL value decides whether the message is printed or not.

Parameters
severityA diag_trace_lvl_t enumeration value specifying the severity of the message.
osi_stackA reference to the ost_stack_t that triggered the diagnostic message. This parameter can be NULL.
layerA diag_trace_src_t enumeration value specifying the source of the diagnostic message.
fmtA printf-like format string specifying the diagnostic message. This parameter is passed as a null-terminated string of single-byte characters. The value must not be NULL.
Returns
0 (zero) if the printing of the message was skipped because of the severity value; otherwise, the number of characters that were printed to the standard output stream.

The DIAG_PRINTF function first evaluates whether the severity against the value of the global TRACE_LEVEL variable value. If severity is greater than TRACE_LEVEL the diagnostic message will not be printed to the standard output stream. Only if it is below or equal, the message will be printed.

Variable Documentation

§ TRACE_LEVEL

diag_trace_lvl_t TRACE_LEVEL

Application-global variable that specifies the maximum severity level of printed diagnostic messages.

The DIAG_PRINTF function is designed in such a way, that diagnostic messages whose severity value is higher than the TRACE_LEVEL global value will not be printed to the output stream.
