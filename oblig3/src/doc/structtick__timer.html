Detailed Description

Type representing a tick-timer object.

A timer object has expired when its tick_count value has become equal to or greater than its target_ticks value.

Field Documentation

§ callback

tick_timer_callback_t callback

The callback function to invoke when the timer object expires.

A reference to the callback function that is invoked when the timer object expires, or NULL if no callback is specified.

The callback function is invoked with the ctx set to the ctx field value of the timer object that expired.

§ ctx

void* ctx

The context reference for the timer object.

A generic reference that is passed on to the callback function when the timer object expires.

It is up to the implementation of the callback function to decide the actual value of this reference.

§ target_ticks

int target_ticks

The targeted number of ticks that is required for this timer to expire.

A 32-bit signed integer value greater than 0 (zero).

A timer object has expired when its tick_count value has become equal to or greater than its target_ticks value.

Setting this field to 0 (zero) or less, will effectively cause the timer object to expire immediately.

§ tick_count

int tick_count

The number of ticks that the timer object has ticked since it has been set.

A value between 0 (zero) and target_ticks.

A timer object has expired when its tick_count value has become equal to or greater than its target_ticks value.
